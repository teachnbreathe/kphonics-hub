먼저 `SPEC.md`의 §20 "사이트워드 섬 (Sight Word Island)"과 `K-Phonics_OPS_Adventure_지도체계_마스터표.md`의 §4를 읽고 아래 작업을 진행해줘.

## 배경

지금 K-Phonics OPS™: Adventure는 Chapter 1(Sound Seeds)이 구현 완료 상태고, 사이트워드는 두 갈래로 설계돼 있어:

1. **챕터 내장형 (기존, Chapter1은 이미 구현됨)**: `efl-sight-1/2/3` 같은 스테이지로 규칙 스테이지 사이사이에 삽입, 50단어(🔴12+🟢38), `sightStage()` 헬퍼로 생성. **이건 건드리지 마.**
2. **사이트워드 섬 (신규, 이번에 구현할 것)**: 나머지 83단어를 담는 상시 오픈 별도 화면. 챕터 클리어에 연동된 5개 티어(T1~T5)로 해금되지만, 티어 안에서는 순서 자유. 50단어 예산 제한 없음.

## 구현할 것

### 1. 데이터 (js/data.js)

- `KP_ISLAND_WORDS` 배열 신설. 각 항목: `{ word, kr, emoji, tier, tag }` (tag는 `"green"` 또는 `"red"`, 기존 🟢/🔴 분류와 동일 의미)
- 단어·티어·태그는 SPEC.md §20-3 표(또는 마스터표 §4)에 있는 83개 단어를 그대로 사용 — T1 12개, T2 14개, T3 10개, T4 44개, T5 3개, 합계 83개.
- 각 단어는 기존 하트 단어 카드와 동일하게 그림(이모지)+한국어 뜻을 가져야 함 (EFL 필수 원칙, SPEC §16-1 — 위반 불가).
- 티어 해금 조건은 챕터 클리어 여부로 판단: T1=Chapter1 전체 클리어, T2=Chapter2 클리어 … T5=Chapter5 클리어. (지금은 Chapter1만 존재하니 T1만 실제로 해금되고 T2~T5는 챕터가 구현되는 대로 자연히 열리게 설계할 것 — 하드코딩으로 챕터를 다시 만들지 말 것.)

### 2. 화면/로직 (js/game.js)

- 월드맵에 6번째 특수 섬 아이콘 추가 (기존 5개 챕터 섬과 별도, 항상 클릭 가능).
- 섬 진입 시 티어별 탭 또는 섹션 UI. 해금된 티어는 카드 그리드로 단어 표시, 미해금 티어는 안개/실루엣 처리 + "OO 챕터를 깨면 열려요" 안내.
- 카드 클릭 → 기존 발음 재생 로직(오디오 아키텍처 원칙 준수: 음가 아님, 단어 전체 발음이라 TTS 사용 가능) 재생, 다시 클릭 또는 플립 → 그림 뒷면에 한국어 뜻.
- "학습 완료" 체크 표시(선택, 기존 단어 연습장 `practice-chip` 패턴처럼 로컬 저장) — 이건 진도 게이트가 아니라 순수 자기 체크용.
- 섬은 `S.clearedStages`/`S.gems`/`stage_order` 게이트 시스템과 분리된 화면이므로, 챕터 진행 상태(젬 개수 등) 카운트에 영향 주면 안 됨. 저장은 별도 키(예: `S.islandWords = { [word]: { mastered: bool } }`)로 관리.

### 3. 스타일 (css/style.css)

- 기존 `.heart-grid`/`.heart-card`류 패턴을 참고해서 섬 전용 그리드·카드·미해금 안개 효과 스타일 추가. 크고 명확한 디자인 원칙 유지.

### 4. 검증

- `node --check js/data.js`, `node --check js/game.js` 통과 확인.
- 83개 단어 개수·티어별 개수(12/14/10/44/3)가 SPEC.md §20-3 표와 정확히 일치하는지 스크립트로 대조.
- 기존 Chapter1 세이브 데이터(로컬스토리지)와 충돌 없는지 확인 — 특히 `S.gems` 카운트 로직(`hudHTML()`)이 섬 단어를 젬 개수에 포함시키지 않는지 확인.

## 하지 말 것

- 기존 `efl-sight-1/2/3` 및 50단어 챕터 내장형 사이트워드는 수정하지 마.
- 섬 단어를 스테이지 게이트(`stage_order`, `KP_STAGES`)에 넣지 마 — 완전히 별도 화면이어야 함.
- Chapter2~5 자체(규칙 스테이지)를 새로 구현하지 마. 이건 섬 UI/데이터 작업만 요청하는 거야. T2~T5는 나중에 해당 챕터가 구현될 때 자동으로 해금 로직만 타면 됨.

작업 끝나면 브라우저에서 `npx serve .`로 로컬 서버 띄워서 실제로 섬에 들어가 카드가 뜨는지, 클릭 시 발음+플립이 되는지 확인해줘.
